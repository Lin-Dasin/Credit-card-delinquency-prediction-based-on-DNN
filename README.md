# Credit-card-delinquency-prediction-based-on-DNN
**SeriousDlqin2yrs**：90 天 + 严重逾期（标签），二分类 (1 = 是 / 0 = 否)

## 📌 数据完整处理流程

### 1. 探索性数据分析（EDA）
- 目的：理解数据分布、缺失模式、异常情况、类别平衡性等，为后续处理提供依据。
- 输出：初步决定变量处理策略（如哪些变量需截尾、填充、变换等）。

### 2. 前置规则型清洗（无数据泄露风险）
- 删除完全重复的样本。
- 删除逻辑错误样本（如年龄为负、收入与支出矛盾等）。
- 删除完全无效的特征（如 ID、纯文本备注、全为缺失的列）。
- **注意**：此步骤基于业务常识，不涉及数据分布，可直接在原始数据上执行。

### 3. 划分训练集 / 测试集
- 使用**分层抽样**（若分类任务）确保训练集和测试集的违约比例与原始数据一致。
- 测试集从此“冷冻”，不再参与任何后续的参数拟合或数据预处理学习。

### 4. 基于训练集的统计型预处理参数学习
> **前提**：所有需要从数据中学习的参数（如均值/中位数填充、分位数截尾阈值、方差阈值等）**必须仅基于训练集**计算，然后应用到训练集和测试集。

- **缺失值填充**：在训练集上计算填充值（如中位数、众数、常数），用这些值填充训练集和测试集。
- **异常值截尾**：若截尾阈值需从数据中确定（如 99% 分位数），则在训练集上计算该阈值，并对训练集和测试集进行截尾。
  - **特别说明**：若截尾阈值是**业务固定值**（如 DebtRatio > 10 截尾为 10），则此步骤可提前至第 2 步（规则型清洗）或在此步直接应用固定规则，无需从训练集学习。
- **特征筛选**：若基于统计量（如方差、缺失率）筛选特征，则在训练集上确定保留的特征列表，并应用到训练集和测试集（剔除相同特征）。

### 5. K 折交叉验证（仅在训练集上进行）
- 将训练集划分为 K 折（例如 5 折），保持类别分布（分层）。
- 对每一折 **i**：
  1. **拆分**：当前折的数据分为训练部分（K-1 折）和验证部分（1 折）。
  2. **标准化**：在训练部分上拟合标准化器（如 StandardScaler），然后用该标准化器变换训练部分和验证部分。
  3. **过采样**（若需处理类别不平衡）：仅在训练部分上进行过采样（如 SMOTE），验证部分保持原始分布。
  4. **模型训练**：在过采样后的训练部分上训练模型，并用验证部分评估性能（记录 AUC、F1 等指标）。
- 汇总 K 折结果，得到模型在训练集上的泛化能力估计，用于调参或模型选择。

### 6. 最终模型训练
- 使用**整个训练集**重新拟合预处理参数：
  - **标准化**：在整个训练集上拟合标准化器，然后变换整个训练集和测试集。
  - **过采样**（若交叉验证中使用了过采样）：对整个训练集进行过采样，保持与交叉验证一致的平衡策略。
- 在过采样后的整个训练集上训练最终模型。

### 7. 测试集评估
- 用“冷冻”的测试集（已应用第 4 步的填充、截尾、特征筛选，以及第 6 步的标准化）对最终模型进行预测，得到泛化性能指标。
- 此结果为模型在真实分布上的最终表现。

## 模型选择

### 对比模型
- DNN（主模型）：深度神经网络 (3 个隐藏层)
- Traditional NN：传统浅层神经网络 (1个隐藏层)
- Decision tree (J48) ：决策树
- Logistic regression： 逻辑回归
